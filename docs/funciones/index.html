    Funciones

    Es muy fácil declarar una función en Swift. Sólo hay que usar la palabra reservada func:

    func greet() {
    print("Hello!")
}

    El ejemplo en el listado anterior declara una función llamada greet, que no acepta ningún parámetro, y al ser invocada imprimirá en pantalla el texto Hello!.

    Para invocar una función hay que escribir su nombre seguido de paréntesis:

    greet() // #=> Hello!

    Muchas de las funciones que escribas en un futuro trabajarán con algún tipo de información. Esta información puede ser proporcinoada a la función como parámetro de la misma:

    func greet(name: String) {
	print("Hello, \(name)!")
}

greet(name: "Oscar") // #=> Hello, Oscar!

    Una función puede tener tantos parámetros como se desee, pero es recomendable mantener la sanidad al momento de crear funciones. Una función con muchos parámetros usualmente es un indicio de que el código puede ser refactorizado.

    Argumentos etiquetados

    func add(number firstNumber: Int, and secondNumber: Int) {
	print(firstNumber + secondNumber)
}

add(number: 40, and: 50) // #=> 90

    Swift es un lenguaje que se preocupa mucho porque el código sea lo más expresivo posible. Para ese efecto, se puede adornar los parámetros de las funciones con nombres externos y nombres internos.

    En el ejemplo anterior, se le asignaron etiquetas a los parámetros firstNumber y secondNumber. Estas etiquetas sirven para que la llamada a la función sea mucho más expresiva sin sacrificar facilidad de uso dentro de la misma.

    Si no se proporciona una etiqueta explícitamente, como en la función greet(name:), la etiqueta y el parámetro se llamarán igual.

    Del mismo modo, se puede obligar a Swift a omitir por completo las etiquetas en las llamadas a las funciones:

    func add(_ firstNumber: Int, and secondNumber: Int) {
	print(firstNumber + secondNumber)
}

add(40, and: 50) // #=> 90

    En casos como el anterior, remover la etiqueta del primer parámetro de la función resulta en que la llamada a la misma sea mucho más expresiva, pues number resultaba realmente excesivo.

    Las etiquetas en las funciones existen para mejorar la legibilidad, y deben usarse con criterio, pues tanto abusar de ellas como removerlas por completo puede afectar severamente la facilidad de mantenimiento de código.

    “Claridad sobre brevedad” es la regla que se debe tener en mente.

    Retorno de valores

    Una función también puede retornar valores. Cuando se espera que una función retorne algún tipo de dato, se debe declarar explícitamente:

    func double(_ number: Int) -> Int {
	return number * 2
}

print(double(2)) // #=> 4

    Una función puede retornar cualquier tipo de dato.

    El ejemplo anterior toma un número entero y lo multiplica por dos, regresando el resultado de dicha operación. Se dice que la función tiene un tipo de retorno Int, especificado por el -> Int después de la lista de parámetros.

    El resultado de la función double(_:) luego es pasado a la función print(_:), que a su vez lo imprime en pantalla.

    Una función también puede ser pasada como parámetro a otra función, siempre y cuando sus tipos de datos sean compatibles. En un capítulo posterior se verá un ejemplo de esto.

    No es necesario que una función tenga un tipo de dato de retorno para poder usar return. De hecho, si se desea interrumpir la ejecución de una función antes de que ésta ejecute su última linea, se puede hacer simplemente usando return.

    func verify(user: User) {
	if user.verified {
		// El usuario ya ha sido verificado,
		// no es necesario continuar con la ejecución de la función
		return
	}
	
	// Proceder con la rutina de verificación del usuario
}
